古典加密算法: 移位加密, 替换加密 , Classical 是对称机密算法的前身

对称加密体制: 古典加密算法的扩展, 私密秘钥 既加密也解密

非对称加密体制: 公钥和私钥加解密体制; 公钥加密私钥机密; 私钥加密公钥解密

散列函数: 保证发送消息的完整性。消息的数字指纹; 非可逆性; 数字签名的完整性核心部分。

数字签名: 签名者 -> 签名算法[私钥] -> 数字签名 -> 加密
         认证者 -> 认证算法[公钥] -> 认证成功 -> 解密
         是签名者使用私钥对消息签名后产生的一个自己数组称之为数字签名, 私钥进行签名的算法称之为签名算法, 这种只能有公钥认证的机密方式又叫 单向认证！
         最理想的状态: 甲方(私钥A 公钥B) 乙方(公钥A 私钥B) 这种双方都持有公私钥的认证方式称为双向认证
         甲方发消息给乙方 : 甲方 私钥A 签名 + 加密  ->  乙方 公钥A 解密 + 认证
         乙方发消息给甲方 : 乙方 私钥B 签名 + 加密  ->  甲方 公钥B 解密 + 认证
