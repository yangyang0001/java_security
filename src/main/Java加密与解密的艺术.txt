科克霍夫原则: 消息的安全性不依赖于加密算法, 只依赖于秘钥

古典加密算法: 移位加密, 替换加密, Classical 是对称加密算法的前身

对称加密体制: 古典加密算法的扩展, 私密秘钥 既加密也解密
             对称加密常用算法 AES, DES

非对称加密体制: 公钥和私钥加解密体制; 公钥加密私钥机密; 私钥加密公钥解密
              常见非对称加密算法 RSA, DSA

散列函数: 保证发送消息的完整性。消息的数字指纹; 非可逆性; 数字签名中完整性的核心部分。
         常见散列算法 MD, SHA, Mac

数字签名:
         1.在加密系统模型中的数字签名的基本用法:
            签名者 -> 签名算法[私钥] -> 数字签名 -> 加密
            认证者 -> 认证算法[公钥] -> 认证成功 -> 解密

         2.概念:
            签名者使用私钥对消息签名后产生的一个字节数组称之为数字签名, 私钥进行签名的算法称之为签名算法; 上面单项的签名和认证 称为 单向认证

         3.最理想的状态: 甲方(私钥A 公钥B) 乙方(公钥A 私钥B) 这种双方都持有公私钥的认证方式称为双向认证
            甲方发消息给乙方 : 甲方 私钥A 签名 + 加密  ->  乙方 公钥A 解密 + 认证
            乙方发消息给甲方 : 乙方 私钥B 签名 + 加密  ->  甲方 公钥B 解密 + 认证

公钥基础设施:
    I.  PKI(Public Key Infrastructure)
        1.认证中心  2.数字证书仓库  3.数字销毁和备份  4.数字证书撤销  5.API
            a.认证中心包括: 注册服务器, 注册公司, 注册中心服务器, 作用: 证书发放, 证书撤销, 证书更新, 证书验证

    II. 数字证书
        结构: ID, 公钥, 颁发者, 数字签名等
        四类: X.509公钥证书, SPKI(Simple Public Key Infrastructure), PGP(Pretty Good Privacy), Attribute证书
